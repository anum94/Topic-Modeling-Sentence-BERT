Job Description Framework

Here you can find some information about how this platform works. The framework includes 3 tabs whose purposes are defined below.

Home

This tab provides information about navigating through the framework and an overview of available functionalities

Job Analytic

This tab provides the option select a visualization, Model, and other parameters such as Number of Clusters and N-gram.

1) Job Cluster:

This paramter defines the number of groups in which the jobs should be clustered. This can your assumption and a good value can be found through trial and error.

For example, if the data contains jobs from 4 departments, it might be a good idea to set cluster value to 4.

It is recommended to try different options to see which one gives best results.

To use this tab, the results should be precomputed through Precompute Data tab. However, For the ease of user, some initial results have been precompute

2) N-grams:

When picking topic words for each cluster, this parameter lets you decide the n-gram for the words.

N-gram = 1 would give you words such as "Talent" , "acquisition"; whereas N-gram = 2 would give you words like "Talent-Acquisition".

3) Model:

Offers different choice and also the combinations of models to use for clustering the jobs.

4) Visualizations:

Various types of visualizations are available. Visualization descripted is displayed along with each analytic.

NOTE: Only the precomputed results can be seen on this tab.

Precompute Data

Since running the algorithm can take some time, there is a separate tab to train all models and precompute the results beforehand for faster loading when doing analytics.

A new data file (csv or excel) can be uploaded in this tab.

The data file should contain the following columns 'Id', 'Title', 'FullDescription' .

Configurations for which the results should to be precomputed can also be set there.
